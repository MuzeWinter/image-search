# Claude Code 任务单：Phase 3 — 资料库管理 + 文件扫描

## 角色

你是本项目的代码实现 Agent，遵守项目全部规则文件。

## 本次目标

实现资料库管理页面 + 文件扫描引擎。用户可添加/删除资料库文件夹，触发扫描，实时查看进度。

## 当前已有基础

- Phase 2 完成的服务注册中心、数据库、设置持久化
- 资料库 CRUD 服务已可用
- `pages/Library.tsx` 骨架页面

## 必须实现

### 1. 资料库管理页面
- 显示已有资料库列表（路径、文件数、上次扫描时间、状态）
- "添加资料库"按钮：打开文件夹选择对话框
- 每个资料库行：打开文件夹 / 重新扫描 / 删除 按钮
- 扫描状态指示：idle（灰点）/ scanning（蓝旋转）/ ready（绿点）/ error（红点）

### 2. 文件扫描引擎
- Python `backend/services/scan_service.py`:
  - 递归扫描目标文件夹
  - 文件类型识别（按扩展名）
  - 文件去重（SHA256 hash）
  - 分类统计（Excel/图片/CAD/PDF/其他）
  - 支持中断扫描
- 扫描进度通过 Tauri event 实时推送到前端

### 3. 扫描进度显示
- 非模态进度条内嵌在页面中
- 显示：当前文件路径、已处理数/总数、百分比
- "暂停" / "继续" / "取消"按钮

### 4. Excel 解析
- Python `backend/services/excel_service.py`:
  - 打开 xlsx/xls/xlsm 文件
  - 提取所有工作表数据
  - 检测并提取内嵌图片（openpyxl）
  - 生成 IMG 编号（IMG-000001 格式）
  - 写入 images 表

### 5. 扫描报告页面
- `pages/ScanReport.tsx`: 真实数据
  - 扫描历史列表
  - 每次扫描的新增/修改/删除/移动/错误统计
  - 耗时显示

### 6. 变更检测
- 比较文件 hash 检测：新增、修改、删除、移动、改名
- 写入 change_logs 表
- `pages/Changelog.tsx`: 真实变更日志

## 禁止事项

- 不许删原有文件
- 不许用 mock 数据
- 不许 git commit/push
- 扫描不卡 UI

## 验收标准

- 可添加资料库文件夹
- 可触发全量扫描
- 扫描进度实时更新在页面中
- 扫描完成后自动写入数据库
- Excel 内嵌图片被提取并生成编号
- 扫描报告显示正确统计
- 可中断扫描
