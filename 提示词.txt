请开发一个 Windows 桌面软件，软件名称为“设计资料图片检索助手”。

这个软件用于产品设计师管理多年设计资料。用户设置一个或多个目标资料文件夹后，软件自动递归扫描文件夹内的 Excel 表格、Excel 表格中粘贴/插入的图片、目录下所有独立图片、UG/NX 图纸文件、辅助 CAD 文件、PDF 文件、渲染图、客户参考图、设计草图、产品照片等资料，并建立本地视觉检索资料库。

软件核心目标：用户拿到客户发来的一张图片后，可以直接拖入、粘贴、截图或选择图片进行搜索。软件使用本地 AI 图片检索模型，在历史资料库中搜索相似图片，并返回对应的图片来源、Excel 表格位置、UG/CAD 文件路径、PDF 路径、原始图片路径、编号和相关备注。

软件必须本地运行，不上传客户照片、Excel 表格、UG 图纸、图片文件、PDF 文件和文件路径到云端。

一、软件定位

软件不是单独针对 UG 文件的检索工具，而是一个“本地设计资料视觉检索软件”。

它需要统一管理和搜索以下资料：

1. Excel 表格中直接粘贴或插入的图片。
2. Excel 单元格中记录的图片路径。
3. 目标目录下所有独立图片文件。
4. 产品渲染图。
5. 客户参考照片。
6. 产品实物照片。
7. 设计草图。
8. 软件截图。
9. PDF 首页或页面预览图。
10. UG/NX 文件。
11. STEP、IGES、JT、DWG、DXF 等辅助 CAD 文件。
12. 用户手动补充的图片。

搜索逻辑是：

客户照片
↓
本地 OpenCLIP 提取图片特征
↓
FAISS 搜索全部历史图片资源
↓
命中 Excel 内嵌图片 / 独立图片 / PDF 预览图 / 渲染图 / 参考图
↓
显示该图片关联的 Excel 记录、UG/CAD 文件、PDF 文件或原始图片路径

二、技术架构要求

1. 桌面端使用 Tauri + React + TypeScript，或者 Electron + React + TypeScript。
2. 本地后端使用 Python。
3. 图片检索模型使用 OpenCLIP。
4. 向量检索使用 FAISS。
5. 本地数据库使用 SQLite。
6. OCR 使用 PaddleOCR，可在设置中开启或关闭。
7. 软件需要能打包成 Windows exe。
8. 用户安装后直接可用，不需要自己安装 Python。
9. 软件支持离线使用。
10. 客户图片、Excel、UG/CAD、PDF、图片路径全部在本地处理，不上传云端。
11. UG、STEP、PDF 等文件不需要强制解析三维模型，软件优先通过文件路径、文件名、同目录关系、Excel 记录、预览图片和用户绑定关系建立关联。
12. OpenCLIP、FAISS、SQLite、PaddleOCR 均应随软件本地部署或提供本地模型路径配置。
13. 软件要适合不懂编程的用户使用，界面简洁，操作步骤少。

五、统一编号规则

软件需要为不同资源生成唯一编号。

1. 图片资源编号：
   格式：IMG-000001

图片资源包括：

* Excel 内嵌图片
* 独立图片文件
* PDF 预览图
* 用户手动补图
* 搜索历史截图
* 其他可检索图片

2. Excel 记录编号：
   格式：EX-000001

Excel 记录是指 Excel 表格中的有效行记录，尤其是含有图片、编号、名称、备注、UG 文件名、路径等信息的行。

3. CAD 文件编号：
   格式：CAD-000001

CAD 文件包括：

* UG/NX 的 .prt 文件
* STEP
* IGES
* JT
* DWG
* DXF
* SolidWorks 等辅助 CAD 文件

4. PDF 文件编号：
   格式：DOC-000001

5. 所有编号必须长期稳定保存。

6. 重新扫描时不能随意改变已有编号。

7. 文件移动、改名、内容变化时，应尽量保留原编号，并更新路径和状态。

8. 人工确认过的匹配关系不能被重新扫描自动覆盖。

七、Excel 表格解析功能

软件必须读取目标文件夹内所有 Excel 表格。

2. 自动读取单元格文字。
3. 自动提取 Excel 表格中粘贴或插入的所有内嵌图片。
4. 自动判断每张图片所在的工作表、所在行、所在列和对应单元格区域。
5. 自动读取图片所在行的编号、名称、客户、类别、备注、UG 文件名、图片路径、文件路径等字段。
6. 如果 Excel 表格列名不固定，需要通过列名关键词识别。
7. 如果没有明确列名，也要保存行号、文字内容和截图。
8. 每条有效 Excel 记录生成唯一编号 EX-000001。
9. Excel 内嵌图片提取后，也必须生成 IMG 编号。
10. Excel 内嵌图片必须加入 OpenCLIP + FAISS 图片检索索引。
11. 用户上传客户照片搜索时，搜索范围必须包括 Excel 表格内嵌图片。
12. Excel 中如果存在图片路径，也要把对应图片加入图片资源库。
13. 同一行有多张图片时，必须全部提取，并允许同一个 EX 记录关联多张 IMG 图片。
14. 如果某张 Excel 图片无法准确判断所在行，也要提取并加入图片库，同时标记为“待确认”。
15. 提取 Excel 图片时不能修改原始 Excel 文件。
16. 提取出来的 Excel 图片保存到软件缓存目录，例如 thumbnails/excel_images/。
17. 重新扫描 Excel 时，需要检测表格内嵌图片是否新增、删除、移动或变化。
18. 如果 Excel 图片位置变化，但图片 hash 相同，应判断为“图片移动”。
19. 如果 Excel 图片 hash 变化，应判断为“图片修改”。

十、图片与 CAD / Excel / PDF 自动关联规则

1. IMG 图片资源。
2. EX Excel 记录。
3. CAD 文件记录。
4. PDF 文件记录。

自动关联优先级：

1. Excel 中明确写了 UG/CAD 文件名或路径，优先匹配。
2. 图片文件名与 CAD 文件名完全一致，优先匹配。

5. Excel 名称匹配 CAD 文件名。
6. Excel 备注关键词匹配 CAD 文件名。
7. 图片与 CAD 文件在同一文件夹，优先匹配。
8. 图片与 CAD 文件在同一子文件夹，次级匹配。
9. 图片与 CAD 文件修改时间接近，可作为辅助判断。
10. PDF 文件名与图片文件名相似时，建立关联。
11. 全资料库模糊匹配。
12. 人工确认匹配。

匹配状态包括：

1. 已确认。
2. 自动匹配。
3. 疑似匹配。
4. 未匹配。
5. 人工绑定。
6. 忽略。

软件必须支持：

1. 一条 IMG 绑定多个 CAD 文件。
2. 一条 EX 绑定多个 CAD 文件。
3. 一个 CAD 文件关联多张图片。
4. 一个 Excel 记录关联多张图片。

软件可以启用 OCR。

OCR 用途：

1. 识别客户照片上的编号、型号、文字。
2. 识别历史图片上的文字。
3. 识别 Excel 内嵌截图上的文字。
4. 识别独立图片上的文字。
5. OCR 结果用于辅助搜索 Excel 记录。
6. OCR 结果用于辅助搜索图片文件名、CAD 文件名和备注。
7. OCR 结果保存到数据库，可用于文字搜索。

OCR 功能可以在设置中开启或关闭。
